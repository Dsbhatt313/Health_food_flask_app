About the App

Introduction

This application is designed to help users calculate their daily calorie needs based on their personal information, activity levels, and meal preferences. It also assists in finding recipes that match the user's input ingredients and meet their calorie requirements.

Features

  • Calorie Calculation: Using user-provided data such as age, gender, height, weight, and activity level, the app calculates the Basal Metabolic Rate (BMR) and the total daily calorie needs.
  • Recipe Matching: Based on the input ingredients, the app searches for recipes that fit within the user's meal calorie limits and displays them.
  • Activity and Meal Tracking: Users can select their daily activities and meal types, which the app uses to tailor the calorie recommendations.
  • Database Integration: The app stores user input data and retrieves nutritional information from an external database.

How It Works

The app collects user input through a multi-step form. Each section of the form gathers specific data about the user:

  1. Select Activities: Users choose their daily activities, which help determine their activity level.
  2. Activity Duration: The duration of each activity is selected, contributing to the overall activity level calculation.
  3. Personal Information: Users provide their gender, age, height, and weight.
  4. Meal Type and Preferences: The app asks for meal types, age intervals, and any medical issues that may affect diet.
  5. Ingredient Input: Users can enter ingredients they have, and the app will find matching recipes.

Based on the provided information, the app calculates the user's calorie needs and suggests recipes that fit their dietary requirements.

Technologies Used

  • Flask: A lightweight Python web framework used to build the server-side of the application.
  • HTML, CSS, and Bootstrap: Used for structuring and styling the web pages, ensuring a responsive and user-friendly interface.
  • JavaScript: Enhances the user experience by enabling dynamic content and form validation.
  • MySQL: A relational database used for storing user data and nutritional information.
  • Pandas: A Python library for data manipulation and analysis, used for reading and processing the nutritional data from an Excel file.
